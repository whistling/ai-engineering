# LLaVA-OneVision: 在同一模型中处理单图、多图与视频

> 在 LLaVA-OneVision（Li 等，2024 年 8 月）之前，开源 VLM 世界存在独立谱系：用于单图的 LLaVA-1.5、多图模型如 Mantis 和 VILA、视频模型如 Video-LLaVA 与 Video-LLaMA。各自能在自己的基准上取胜，但在其它场景表现欠佳。LLaVA-OneVision 提出可以通过单一课程训练一个模型统治这三类场景，并且出现的任务迁移效应（单图技能传导到视频、多图推理增强单图能力）胜过专门化模型的简单加和。配方看似简单：跨场景保持恒定的视觉令牌预算，再加上从单图到 OneVision（多图）到视频的显式课程。本讲读的是预算、课程与出现的行为。

**Type:** 构建  
**Languages:** Python（标准库、令牌预算求解器 + 课程规划器）  
**Prerequisites:** Phase 12 · 05（LLaVA）、Phase 12 · 06（任意分辨率）  
**Time:** ~180 分钟

## 学习目标

- 设计一个在单图、多图和视频输入之间保持恒定的视觉令牌预算。  
- 排列训练课程，使技能从单图迁移到视频且不会出现灾难性遗忘。  
- 解释为何在正确的课程下单一模型能在相同参数量下胜过专门化模型。  
- 说出 LLaVA-OneVision 报告的三项涌现能力：多摄像头推理、编号标记提示、iPhone 截图代理。

## 问题背景

图像、多图和视频对模型的要求各不相同。

单图需要高分辨率令牌（AnyRes，约 2880 个视觉令牌）以捕捉 OCR 与细节。每个样本预算：一张图，2880 令牌。

多图需要多张中等分辨率的图像（每张约 576 令牌），以便跨图推理能放进上下文。每个样本预算：4–8 张图，每张 576，合计 2300–4600 令牌。

视频需要许多低分辨率帧（池化后每帧约 196 令牌）以捕捉时间动态。每个样本预算：8–32 帧，每帧 196，合计 1600–6200 令牌。

若训练独立模型，你可以为某一场景选择预算；若训练单一模型，则需要预算在各场景间合理缩放，而不至于打爆上下文窗口。

在 OneVision 以前，默认策略是“只训练一个场景，忽略其它场景”。Video-LLaVA 是在图像模型上通过额外训练阶段改造得到的视频能力。LLaVA-NeXT 通过切片（tiling）添加了多图支持。没有模型能将三者干净利落地整合在一起。

## 概念

### OneVision 令牌预算

LLaVA-OneVision 选定了大约 3000–4000 视觉令牌/样本的统一预算，并在不同场景中以不同方式分配：

- 单图：AnyRes-9（3x3 切片 + 缩略图），每个切片以 384 分辨率对应 729 个 patch，使用激进的双线性（bilinear）2x2 池化 → 每片 182 令牌。总计：9 * 182 + 182 = 1820 令牌。或使用 AnyRes-4，切片为 729/token = 2916 + 729。  
- 多图：每张图为中等分辨率（384，无切片），729 令牌/张。6 张图 → 4374 令牌。  
- 视频：32 帧，384 分辨率并使用激进的 3x3 双线性池化 → 每帧 81 令牌。总计：32 * 81 = 2592 令牌。

该分配在总令牌数上保持大体恒定。LLM 不会看到超出其上下文窗口的批次。编码器会在各场景下产生不同几何形状，但 LLM 消耗的是相同的预算。

### 三阶段课程

LLaVA-OneVision 采用三阶段训练：

1. 单图 SFT（阶段 SI）。所有数据为单图 + 文本。使用高分辨率 AnyRes 输入训练，教授感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加上 OneVision 专用的单图数据。  
2. OneVision SFT（阶段 OV）。混合单图、多图与视频（视频采样为均匀帧）。在统一令牌预算下训练，教会模型处理异构的批次形状。不断权重（不重置），接续自阶段 SI。  
3. 任务迁移（阶段 TT）。继续以目标任务混合进行训练，通常根据产品需求更偏重多图或视频。可选的部署微调。

关键：课程顺序很重要。若先训练视频或多图，再训练单图，图像性能会更差，即便数据相同。论文做了明确消融验证这一点。

### 为什么课程有效

单图训练构建了感知基础。patch 令牌承载细粒度视觉特征；LLM 学会将其与文本整合。多图和视频引入结构性挑战（哪张图是哪一张、先后发生了什么），没有强感知基础这些结构很难学会。

若从头同时训练所有场景，模型对感知欠拟合（每个批次中的单图数据被稀释），而对结构过拟合（多图/视频数据比例较大）。结果是模型学会跨图推理模式但视觉部分浅薄。

课程顺序先给出感知强度（阶段 SI），再加入组合性/时间推理（阶段 OV），从而保留两者能力。

### 跨场景涌现技能

LLaVA-OneVision 报告了三项涌现能力：

1. 多摄像头推理。分别在多图与视频上训练；推理时在多摄像头驾驶场景下询问，模型能正确整合视角，尽管训练时未见过该具体格式。  
2. 编号标记提示（set-of-mark prompting）。用户在图像上用编号标记对象；模型能回答“标记 3 相对于标记 7 在做什么”。训练时既没有标注标记也没有专门的注记；这是空间定位 + 多图参考组合所学的结果。  
3. iPhone 截图代理。用户给出 iPhone 屏幕截图并要求规划下一步点击。模型在 UI 截图、用户工作流视频和多图前后对比数据上训练，从而泛化到该代理用例。

这些并非直接训练的任务；它们从课程的组合结构中涌现出来。

### 视觉令牌池化

令牌预算要求必须做池化。OneVision 在 2D patch 网格上使用双线性插值：例如 24x24 = 576 patches，变为 12x12 = 144（2x 因子）或 8x8 = 64（3x 因子）。池化在 patch 网格空间进行，而非在 token 空间，以保留局部性。

每个场景选择的池化因子本身就是超参数。更少的池化 = 更多令牌 = 更丰富的表示；更多的池化 = 更少令牌 = 能装下更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续（LLaVA-OneVision-1.5，arXiv:2509.23661）在训练数据、模型权重与代码上“完全开源”。在某些基准上匹配了专有模型的差距并将该配方民主化。相同课程、更多数据、更好的基础 LLM。无架构变更。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（Lesson 12.09）做了不同选择。它使用 M-RoPE 和动态 FPS 而不是固定池化。它的预算随输入而扩展 —— 1 分钟视频消耗的令牌比 5 秒视频多。LLaVA-OneVision 固定预算并通过池化缩放输入。两者都能工作；它们在可配置性与可预测性之间权衡。

## 使用方法

`code/main.py` 是一个用于 OneVision 风格 VLM 的课程与预算规划器。给定每样本令牌预算和目标场景分布（例如 40% 单图、30% 多图、30% 视频），它会：

- 为各场景分配分辨率、池化因子与帧数。  
- 检查每个场景是否适配共享预算。  
- 报告预期令牌数、LLM FLOPs 以及哪些场景令牌不足。  
- 打印逐阶段的训练日程。

用它来规划 OneVision 风格的微调，或对 VLM 部署的每次请求成本做合理性检查。

## 交付物

本课产出 `outputs/skill-onevision-budget-planner.md`。给定目标任务分布和每样本预算，它会输出 AnyRes 因子、每帧池化、视频帧数与课程阶段权重。在训练或微调统一场景 VLM 时请使用此工具。

## 练习

1. 你的产品支持 80% 单图、10% 多图（2–4 张图）、10% 视频（8–16 帧）。设计令牌预算。你会把不需要重度多图的节省预算放到哪里？  
2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出第四项可能会被课程解锁但论文未报道的涌现技能。  
3. 更换课程顺序——先训练多图，再单图，最后视频。预测哪些基准会退化以及原因。  
4. 论文报告的视频基准仅用每样本 8 帧进行训练。这个设置能在推理时泛化到 30 秒视频吗？首先被打破的是令牌预算还是时间推理？  
5. 将 24x24 patches 做双线性池化到 12x12 是每维 4x 的缩减。在标准库 Python 中实现该池化，并验证每个 2x2 块的均值是否匹配双线性输出。

## 关键术语

| 术语 | 常说的说法 | 实际含义 |
|------|-----------|---------|
| OneVision scenario | “单图、多图或视频” | 统一 VLM 处理的三种输入形状之一；预算在它们之间保持恒定 |
| Token budget | “每个样本多少令牌” | LLM 在每个训练/推理样本中看到的总视觉令牌，通常为 3000–4000 |
| Curriculum | “训练顺序” | 选择的阶段排序（单图 → 多图 → 视频），以实现涌现迁移 |
| Bilinear pooling | “令牌压缩” | 在 patch 网格（2D）上应用双线性插值以减少令牌数同时保留局部性 |
| Emergent skill | “未专门训练但能用” | 因课程组合而在推理时出现的能力，无需匹配的训练数据 |
| AnyRes-k | “k 切片设置” | k 个固定分辨率子切片加一个缩略图，典型 k ∈ {4, 9} |
| Task transfer | “跨场景泛化” | 在单图上学到的技能通过共享骨干应用到视频（反之亦然） |
