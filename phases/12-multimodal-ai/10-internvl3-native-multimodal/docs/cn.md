# InternVL3: 原生多模态预训练

> 在 InternVL3 之前，每一个开源 VLM 都遵循相同的三步配方：先训练一个在万亿级文本上训练的文本 LLM，接着加上视觉编码器，最后对接口部分进行微调。这种方法可行，但会产生对齐债务 — 文本 LLM 在纯文本上的预训练预算已经花光，并不能原生理解视觉 token。当你事后加入视觉输入时，LLM 必须在不遗忘文本能力的前提下重新学习如何将视觉输入与其文本推理关联起来。InternVL3（Zhu 等，2025 年 4 月）拒绝事后拼接的方法：一次性预训练，从第一步起就交叉包含文本与多模态。结果在 78B 参数的开源模型上，在 MMMU-Pro 上匹配 Gemini 2.5 Pro。本课阅读原生预训练的论据以及实施后会发生的变化。

**Type:** 学习  
**Languages:** Python（stdlib，训练语料混合器）  
**Prerequisites:** Phase 12 · 05，Phase 12 · 07（配方）  
**Time:** ~120 分钟

## 学习目标

- 解释为什么事后拼接的 VLM 训练会积累对齐债务，并列出三个可测量的症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生多模态预训练语料混合及为何 text : interleaved : caption 的比例重要。
- 比较 V2PE（variable visual position encoding）与 Qwen2-VL 使用的 M-RoPE。
- 说出 Visual Resolution Router (ViR) 和 Decoupled Vision-Language (DvD) 的部署优化要点。

## 问题所在

事后拼接（post-hoc）VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics — 它们都采用已经预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）并加上视觉模块。常见的训练阶段通常是：

1. 冻结的 LLM + 冻结的视觉编码器 + 可训练的投影器，在配对描述（caption）数据上训练以对齐嵌入。
2. 解冻 LLM，在指令数据上训练（如 LLaVA-Instruct、ShareGPT4V）。
3. 可选的任务专用微调。

对齐债务表现为三个症状：

- 灾难性遗忘。事后拼接的 VLM 会遗忘纯文本技能。GSM8K 分数下降 5–10 分。Hellaswag 分数下滑。纯文本代理退化。
- 答案漂移。相同视觉问题的细微措辞变化会得到不同答案。视觉编码器与 LLM 的连接绑定比 LLM 自有 token 的绑定更弱。
- 视觉-文本不一致。VLM 可能先正确描述图像，然后在回答问题时给出与自身描述矛盾的答案。视觉 token 无法像文本那样参与 LLM 的内部一致性检查。

这些症状已有充分文献记载。MM1.5 第 4 节对它们做了量化。LLaVA-OneVision 的消融试验也暗示了这些问题。原生预训练是解决方案。

## 概念

### 原生多模态预训练

InternVL3 从头开始训练，语料从一开始就是原生多模态。语料混合为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 配对图像-描述（image-caption）数据
- 5% 视频-文本数据

视觉 token、文本 token 以及跨模态交互从第一步梯度开始就参与同一损失。不需要对齐预训练、没有投影器冻结阶段，也不需要去恢复灾难性遗忘。

基础模型采用单阶段训练。随后再进行指令微调，但基础模型已经把视觉 token 视为一等公民。

### V2PE（variable visual position encoding）

Qwen2-VL 使用 M-RoPE 并进行固定轴分配。InternVL3 引入 V2PE：位置编码根据模态类型（文本、图像、视频）而变化，并带有可学习的缩放。在实践中：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行、列）。
- 视频帧获得 3D 位置（时间、行、列）。

三者共享相同的 RoPE 频率基础，但每个波段的隐藏维度分配是可学习参数，而不是固定切分。允许在预训练期间在时间分辨率和空间分辨率之间权衡。

V2PE 的消融结论：在相同计算量下，视频基准上比 M-RoPE 提升 1–2 分。不是革命性的提升，但更为干净优雅。

### Visual Resolution Router (ViR)

部署优化。并非所有图像都需要全分辨率编码。一张含单一目标且细节较少的照片在 1280px 原生编码时浪费大量 token。ViR 是一个小分类器，在编码前预测回答该问题所需的最小分辨率。

路由分为三档：低分辨率（256 token）、中等（576）、高（2048+）。在生产流量中，60% 的查询用低或中等分辨率就足够。净效果：在质量相同的情况下吞吐量提升 2–3 倍。

### Decoupled Vision-Language 部署（DvD）

在服务大型 VLM 时，视觉编码器对每张图像只运行一次，而 LLM 则会为每个输出 token 进行自回归运算。两者的瓶颈不同（视觉 = 卷积 + 注意力的 GPU 内存带宽；LLM = KV 缓存）。DvD 将它们拆分到不同 GPU，并通过流式传输连接。

对于 8B + 400M 编码器的模型，DvD 相比同机部署大致能将每节点吞吐量翻倍。

### 单阶段 vs 多阶段 的质量差异

InternVL3 的主要基准论断：在 78B 参数下，单阶段预训练加指令微调的结果在 MMMU-Pro 上可匹配 Gemini 2.5 Pro。在 38B 下可匹配 GPT-4o，在 8B 下领先开源 8B 排行榜。全部基于单阶段预训练 + 指令微调流程。

对齐债务假设是可测量的：与 Qwen2.5-VL-7B 相比，InternVL3-8B 在每单位视觉基准增益所带来的文本基准点数损失（如 MMLU、GSM8K）更少。因为训练是一体化进行的，模型更像一个通用模型，而不是两块拼接起来的模型。

### InternVL3.5 与 InternVL-U

InternVL3.5（2025 年 8 月）扩展了配方。相同的原生预训练方法，更多数据，更多参数。MMMU 指标有增量改善。

InternVL-U（2026）加入了统一生成能力 — 通过在同一骨干之上添加 MMDiT 头实现图像输出。“U”代表“Understanding + generation（理解 + 生成）”，旨在追赶 Transfusion 风格的统一模型（见 Lesson 12.13）。相同的原生预训练骨干同时支持理解和生成头。

### 原生预训练的权衡

原生预训练并非无代价：

- 计算资源。重新从头训练一个 VLM 的成本与训练一个文本 LLM 相当 — 数百万 GPU 小时。事后适配可以复用现成的 LLM 权重，从而节省大部分成本。
- 数据。大规模的交错图文语料稀缺。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。相比之下，纯文本可达 15T token。多模态预训练数据稀缺是一个硬约束。
- 基础 LLM 可复用性。原生预训练放弃了以后直接替换新 LLM 的选项。事后拼接允许只重训练适配器就把 Llama-3.1 换成 Llama-4，从而更灵活。

InternVL3 的赌注是：对齐债务比放弃复用更严重。基准支持这一结论。高昂的生产成本也让未来小型实验室难以廉价复现。出于成本考虑，大多数项目仍会继续使用事后拼接的 VLM。

## 使用说明

`code/main.py` 是一个训练语料混合器和 ViR 路由器模拟器。它可以：

- 接受目标语料混合（%text、%interleaved、%caption、%video），并计算每种模态的预期训练步数。
- 在一批查询上模拟 ViR 路由（分布：50% 低细节、30% 中等、20% 高细节），并报告平均 token 数。
- 给定编码器与 LLM 的 FLOPs，报告 DvD 吞吐量估算。
- 打印事后拼接与原生预训练在参数量、计算量、数据量以及预期对齐债务症状上的并列对比。

## 交付物

本课会生成 `outputs/skill-native-vs-posthoc-auditor.md`。给定一个拟议的 VLM 训练计划，它会审计是否应走原生路径或事后拼接、标记对齐债务风险并推荐语料混合。可在你为新的开源 VLM 项目估算规模并需选择训练策略时使用。

## 练习

1. 估算 InternVL3-8B（原生预训练）与 LLaVA-OneVision-7B（事后拼接）之间的计算差异。GPU 小时的比率大致是多少？是什么造成了差距？
2. InternVL3 报告的混合为 40% 文本 / 35% 交错 / 20% 描述 / 5% 视频。如果你的目标任务以视频为主，提出一个新的比例并说明为何基础模型仍然需要大量的文本和描述数据。
3. 阅读 MM1.5 第 4 节关于遗忘的内容。指出事后拼接训练在哪个具体基准中显示了最大的回退？回退了多少分？
4. ViR 将 60% 流量路由到低分辨率编码。它会把哪些查询误路由（应当用高分辨率但被发送至低分辨率）？提出三种路由失败模式。
5. DvD 将视觉和 LLM 拆分到不同 GPU。在什么样的流量模式下，DvD 会比不上同机部署反而降低吞吐量？

## 关键术语

| 术语 | 常说的说法 | 实际含义 |
|------|------------|----------|
| 原生多模态预训练（Native multimodal pretraining） | “从头一起训练” | 文本 + 图像 + 视频 token 从第 1 步起参与同一损失，而不是事后拼接 |
| 对齐债务（Alignment debt） | “事后拼接代价” | 将视觉接入冻结或已训练的 LLM 所导致的文本能力与答案一致性上的可测回退 |
| V2PE | “可变视觉位置编码” | 按模态分配的可学习位置编码分配；InternVL3 的 M-RoPE 继任者 |
| ViR | “分辨率路由器（Resolution router）” | 在编码前为每个查询选择最小分辨率的轻量分类器，从而节省推理 token |
| DvD | “解耦部署（Decoupled deployment）” | 将视觉编码器放在一块 GPU、LLM 放在另一块，通过流式交接；对大型 VLM 可倍增吞吐 |
| InternVL-U | “统一理解 + 生成（Unified understanding + generation）” | 2026 年的后续工作，为原生预训练骨干增加图像生成头 |
| 交错语料（Interleaved corpus） | “OBELICS / MMC4” | 按自然阅读顺序包含文本与图像的文档；是原生预训练的原料 |
