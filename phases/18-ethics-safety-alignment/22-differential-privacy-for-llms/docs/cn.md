# Differential Privacy for LLMs

> DP-SGD 仍然是标准——在梯度更新中注入噪声能够提供形式化的 (ε, δ) 保证。计算、内存和效用方面的开销显著；参数高效的差分隐私微调（LoRA + DP-SGD）是 2025 年常见的配置（ACM 2025）。两类证据存在冲突：基于 canary 的成员推断（Duan et al., 2024）对语言模型报告了有限的成功；训练数据抽取（Carlini et al., 2021；Nasr et al., 2025）则恢复了大量逐字记忆。解决方案（arXiv:2503.06808，2025 年 3 月）：差异来源于度量对象——插入的 canary 与“最易被抽取”的数据不同。新的 canary 设计使得基于损失的 MIA（无需影子模型）成为可能，并给出了对使用真实数据训练且具有现实 DP 保证的 LLM 的首个非平凡 DP 审计。替代方法：PMixED（arXiv:2403.15638）——在推理时通过对下一个 token 分布的专家混合实现私有预测；DP 合成数据生成（Google Research 2024）。新兴攻击：通过 LLM 反馈的差分隐私反演 —— 置信度分数泄露。

## 问题背景

LLM 会记忆。Carlini et al. 2021 表明生产环境的语言模型会在提示下逐字复现训练文本。差分隐私是形式化的防御：使模型输出对任一单个训练样本的依赖可被证明地不显著。2024–2025 年的证据表明 DP-SGD 是必要的，但部署时的 ε 值可能与假定的威胁模型并不一致。

## 概念

### (ε, δ)-差分隐私

随机化算法 M 是 (ε, δ)-DP，当且仅当对任意两个仅相差一个样本的数据集以及任意事件 S，有：
P(M(D) in S) <= e^ε * P(M(D') in S) + δ。

解释：输出分布在参数 ε 所限定的范围内足够接近，使得任何单个个体的贡献不能被可靠地推断出来，除非以概率 δ 发生。

### DP-SGD

Abadi et al. 2016。标准流程：
1. 采样一个小批量。  
2. 计算每个样本的梯度。  
3. 将每个样本梯度裁剪到阈值 C。  
4. 对裁剪后的梯度求和并加入标准差为 σ * C 的高斯噪声。  
5. 使用带噪和更新参数。

隐私成本由会计器跟踪（Moments Accountant、Rényi DP 会计器）。在 LLM 文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而差异很大；没有普适的“安全”默认 ε。已发表示例在某些 LLM 训练设置下大致在 ε ≈ 1–10，但这些只是说明性而非推荐默认值。较低的 ε 通常需要更多噪声并可能增加效用损失。

### LoRA + DP-SGD

对前沿大模型施加完整 DP-SGD 是不可行的。LoRA（Hu et al. 2022）将梯度更新限制在小型适配器上，减少了每样本梯度存储开销。LoRA + DP-SGD 是 2025 年常见的配置。差分隐私保证适用于适配器；基础模型保持固定。

### 2024–2025 年的冲突

两条证据线：

- **Canary MIA（Duan et al. 2024）。** 在训练数据中插入唯一的 canary，测量成员推断攻击者是否能识别它们。在语言模型上报告了有限的成功，表明 MIA 很难。  
- **训练数据抽取（Carlini 2021，Nasr et al. 2025）。** 用前缀提示模型，测量是否能逐字恢复训练文本。报告了大量记忆，表明在相关意义上 MIA 很容易。

2025 年 3 月的解析（arXiv:2503.06808）：两者测量的对象不同。MIA 问的是“样本 e 是否在 D 中？”针对的是插入的 canary。抽取问的是“我能从 D 中恢复什么？”隐私上重要的是“最易被抽取”的示例；canary 未被针对可抽取性优化，因此会低估风险。

新的 canary 设计：基于损失的 MIA（无需影子模型）。对真实数据并带有现实 DP 保证的 LLM 进行了首个非平凡的 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 在推理时实现私有预测。对下一个 token 分布采用专家混合；每个专家看到训练数据的一部分；聚合时加入噪声以保证 DP。完全避免了 DP 训练。  
- **DP 合成数据生成（Google Research 2024）。** 先用 DP-SGD 对 LoRA 进行微调，采样合成数据，然后在合成数据上训练下游分类器。

两者都通过改变威胁模型来回避完整 DP 训练带来的效用成本。

### 通过 LLM 反馈的差分隐私反演

新兴的 2025 年攻击。利用受 DP 训练的模型的置信度分数作为 oracle 来重新识别个体。即使模型输出本身不泄露，置信度分布也可能泄露信息。

防御：不要暴露置信度，或在暴露前对置信度进行截断/量化。这是超出 (ε, δ)-DP 训练之外的额外要求。

## 使用方法

`code/main.py` 模拟了在一个玩具二分类数据集上的 DP-SGD。你可以扫参数噪声倍数 σ 和裁剪范数 C，并跟踪 (ε, δ) 预算与精度代价。一个“canary 攻击”会插入一个唯一训练样本并衡量在 DP 之前与之后基于对数损失的检测能力。

本课产出 `outputs/skill-dp-audit.md`。给定对某语言模型部署的 DP 声称，它会审计：所报告的 (ε, δ) 值、所用的会计器、MIA 评估协议，以及是否评估了置信度暴露向量。

5. 描绘通过 LLM 反馈的 DP 反演攻击。设计一种限制置信度分数泄露的对策并估算其部署成本。

## 关键词

| Term | What people say | What it actually means |
|------|-----------------|------------------------|
| DP | "（ε, δ）-differential privacy" | 形式化隐私：在相邻数据集变化下输出分布接近 |
| DP-SGD | "noise-injected SGD" | 梯度裁剪 + 高斯噪声添加；标准的 DP 训练算法 |
| LoRA + DP-SGD | "efficient private fine-tune" | 在低秩适配器上进行 DP-SGD；2025 年的标准配置 |
| MIA | "membership inference" | 攻击者判定某个示例是否在训练数据中（成员推断） |
| Canary | "inserted watermark example" | 用于测量差分隐私泄露的唯一训练示例 |
| PMixED | "private inference mixture" | 通过对下一个 token 分布的专家混合实现推理时的差分隐私 |
| DP Reversal | "confidence leakage attack" | 利用模型置信度作为 oracle 进行重新识别的攻击 |
