# 多模态评估

> 训练只是循环的一半。另一半是度量。本课从基元构建三种评估面：作为 R@1、R@5、R@10 报告的图像-标题检索；作为精确匹配准确率报告的视觉问答；以及作为 BLEU-4 报告的图像描述生成。每个指标都是模型输出和一个在几秒内运行的合成评估套件上的函数。

**Type:** 构建  
**Languages:** Python  
**Prerequisites:** Phase 19 第 58–62 课（E 轨基础：encoder、transformer、projection、cross-attention 融合、预训练）  
**Time:** ~90 分钟

## 学习目标

- 从图像与标题的嵌入之间的相似度矩阵计算 Recall@K。  
- 从将（图像，问题）对映射到固定答案词表的模型中计算 VQA 的精确匹配准确率。  
- 在不使用任何外部库的情况下，从生成和参考的标记序列计算 BLEU-4。  
- 针对第 62 课训练的模型，在构建的合成套件上运行上述三种评估。

## 问题背景

很容易在训练损失趋于平稳时就宣告多模态模型训练完成。训练损失衡量的是在训练分布上的拟合；它不能衡量模型是否能在留出批次中对样本进行排序、回答问题或写出人类会接受的描述。三个标准评估面是：

- 检索（R@1、R@5、R@10）。为查询标题构建联合嵌入；按余弦相似度对评估池中的每张图像进行排序；报告匹配图像是否出现在前 1、前 5、前 10 名。对称的（图像到文本）形式以相同方式运行。  
- 视觉问答（精确匹配）。给定（图像，问题），模型输出一个答案标记。精确匹配是每个样本的一位判断：预测答案是否等于参考答案？对评估集取平均。  
- 图像描述（BLEU-4）。生成一个描述。针对参考描述计算 1-gram 到 4-gram 精确率的几何均值，并乘以简洁惩罚（brevity penalty）。多参考是标准形式（一张图像，多条参考描述）。

每个指标都是一个薄函数。本课通过代码实现它们，使得数学变得具体，并且评估面始终处于可控范围。真实基准套件（MS-COCO、VQA v2、GQA、OK-VQA）可以插入相同的函数形状。

## 概念

```mermaid
flowchart TB
  Model[训练好的多模态模型] --> Embed[在评估集上的联合嵌入]
  Embed --> Sim[相似度矩阵]
  Sim --> R1[R@1]
  Sim --> R5[R@5]
  Sim --> R10[R@10]
  Model --> VQA[为每个问题预测答案标记]
  VQA --> EM[精确匹配准确率]
  Model --> Caps[生成的描述]
  Caps --> BLEU[与参考的 BLEU-4]
```

### 从相似度矩阵计算 Recall@K

构建图像与标题嵌入之间形状为 (N, N) 的余弦相似度矩阵。对每一行，将列按相似度降序排序。Recall@K 是对角线所在列索引位于前 K 名位置的行所占的比例。对称的 Recall@K（标题到图像）在转置矩阵上计算。两者都要报告。对于 N=100 的评估，R@1 = 0.6 表示 100 条标题中有 60 条将正确图像检索为第一名。

### VQA 精确匹配

对于每个（图像，问题，答案），对图像进行编码、对问题进行嵌入，通过解码器融合，并读出下一个标记。将预测标记 id 与参考 id 比较；相等则判为正确。在评估集上取平均。真实 VQA 数据集通常为每个问题提供多个人类注释的答案，并使用软准确率公式（如果 10 名标注者中至少 3 人一致则计 1.0，其它情况按比例缩放）；本课为清晰起见使用单答案的精确匹配。

### BLEU-4

```text
BLEU-4 = BP * exp(mean(log p1, log p2, log p3, log p4))
```

其中 `p_n` 是修正的 n-gram 精确率（生成的 n-gram 的裁剪计数，即出现在任一参考中的生成 n-gram 的计数上限，总生成 n-gram 数为分母），`BP` 是简洁惩罚：

```text
BP = 1                如果 生成长度 > 参考长度
   = exp(1 - r/g)     否则，其中 r 是参考长度，g 是生成长度
```

在样本较小且某些 `p_n` 为零的情况下需要平滑。实现采用 Chen 与 Cherry 的“方法 1”（对于任何为零的计数，在分子和分母上各加 1），这是低计数情形下最稳妥的默认做法。

### 合成评估套件

使用与第 62 课中相同的模拟语料模式并用一个留出的随机种子，在内存中构建一个 50 样本的评估套件。套件由三个列表组成：

- `pairs`：50 个 (image, caption_ids) 对，用于检索。  
- `vqa`：50 个 (image, question_ids, answer_id) 三元组。  
- `caps`：50 个 (image, [reference_caption_ids, ...]) 条目，每张图像最多 3 个参考描述。

该套件由随机种子确定并与训练语料留出分离，因此指标在模型从未见过的数据上计算。将套件持久化为 JSON 留作练习（见下）。

| Metric | Range | Random baseline (N=50) |
|--------|-------|------------------------|
| R@1 | 0 到 1 | 0.02 (1 / N) |
| R@5 | 0 到 1 | 0.10 |
| R@10 | 0 到 1 | 0.20 |
| VQA EM | 0 到 1 | 1 / vocab |
| BLEU-4 | 0 到 1 | 很小但非零 |

对于在合成数据上运行 50 步的训练，期望这些指标并不高；期望它们高于随机基线 —— 这是演示要检查的内容。

## 构建实现

`code/main.py` 实现了：

- `recall_at_k(sim_matrix, k)`，返回在 `[0, 1]` 范围内的浮点数，双向（图像到文本与文本到图像）均可。  
- `vqa_exact_match(predictions, references)`，返回整型相等性的平均值。  
- `bleu4(generated, references, smoothing=True)`，支持多参考。  
- `build_eval_suite(seed, n_samples, vocab_size, max_len)`，返回三个确定性的评估列表。  
- `evaluate(model, suite)`，运行所有三种指标并返回一个数字字典。  
- 一个示例：加载第 62 课中初始化好的多模态模型，对其评估，然后训练 50 步再评估，并打印前后指标。

运行方式：

```bash
python3 code/main.py
```

输出：前后度量表显示检索从接近随机向模型学到的信号改善，VQA 超过随机，BLEU-4 也有所提升（合成结构对 4-gram 精确率足够友好）。

## 使用指南

每个指标都可以直接映射到生产基准：

- 检索。MS-COCO 5K 验证集、Flickr30K、ImageNet 零样本都是在相同相似度矩阵上的 R@K 问题。用真实文件替换合成评估，函数签名保持不变。  
- VQA。VQA v2、GQA、OK-VQA 使用相同的精确匹配形状（VQA v2 使用软准确率而不是单答案 EM）。  
- BLEU-4。MS-COCO 描述评估、NoCaps、Flickr30K 描述都使用 BLEU-4 以及 CIDEr 和 METEOR。添加 CIDEr 只需再实现一个函数。

对于真实基准，替换 `build_eval_suite` 为真实加载器并保留函数体即可。数学是基准不可知的。

## 练习

1. 为描述评估添加 CIDEr。CIDEr 对 n-gram 使用 TF-IDF 加权，以奖励信息量更高的标记。  
2. 实现软准确率 VQA：每个问题有多个人工答案，如果任一预测匹配则准确率为 min(human_count / 3, 1)。复现 VQA v2 的评分方式。  
3. 添加对 NaN 安全的 `bleu4` 变体，以在生成序列为空时不崩溃。  
4. 在 R@K 之外计算平均倒数排名（MRR）。MRR 对正确项落在 top K 之外的位置更为敏感；R@K 则只对是否落入前 K 敏感。  
5. 在训练期间对模型的五个检查点（step 0, 10, 20, 30, 40, 50）运行评估并绘制学习曲线。确认指标轨迹与损失轨迹相匹配。

## 关键词

| Term | 意义 |
|------|------|
| R@K | 查询中正确匹配落在前 K 结果中的比例 |
| Exact match | 最简单的 VQA 评分：预测答案等于参考答案 |
| BLEU-4 | 1 到 4 gram 精确率的几何均值，并带有简洁惩罚 |
| Multi-reference | 一张图像接受多条参考描述的评估形式 |
| Held-out | 评估集从与训练语料不相交的种子中采样 |
