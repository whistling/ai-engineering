# 共识与拜占庭容错（BFT）在智能体中的应用

> 经典分布式系统的 BFT 遇上了随机性的 LLM。2025–2026 年涌现了三条研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探针加权每一票；**DecentLLMs**（arXiv:2507.14928）无领导，采用并行工作者提案与几何中位数聚合；**WBFT**（arXiv:2505.05103）将加权投票与层次结构聚类结合，划分 Core 与 Edge 节点。来自 “Can AI Agents Agree?”（arXiv:2603.01213）的诚实实证结果是：即使是标量一致性在今天也很脆弱——单个欺骗性智能体就能破坏一个 Mixture-of-Agents。BFT 是必要的但不充分。本课件构建了一个最小 BFT 协议，注入三种智能体特有攻击（拜占庭谎言、谄媚一致性、相关错误的单一文化），并衡量每种共识变体的应对能力。

**Type:** 学习 + 构建  
**Languages:** Python（标准库）  
**Prerequisites:** Phase 16 · 07（心智社会与辩论）、Phase 16 · 13（共享内存）  
**Time:** ~75 分钟

## 问题

你有 N 个 LLM 智能体，每个都输出一个答案。它们发生分歧。多数投票选错了答案，因为两个智能体高度相关（相同基础模型、相同训练数据、相同失败模式）。第三个智能体偶然以一种新的方式出错——所以多数成为错误的多数。

再加入一个欺骗性智能体：它故意撒谎。或者一个谄媚型智能体：它总是同意最后一个发言的人。在经典 BFT 中，假设拜占庭节点的比例为 `f < n/3` 并且可以任意行为。2026 年的现实是：即便诚实，LLM 节点也存在随机性，不同模型之间有相关性，并且会被彼此的输出影响。你不能把它们当作独立的伯努利投票器。

经典 BFT（PBFT，1999）并不是错误的——它是不完整的。它能处理任意位翻转的情况，但无法处理“因为共享训练数据而产生相同幻觉的三个诚实智能体”。本课程基于 PBFT 的基础，叠加三种 2025–2026 年的适配。

## 概念

### 经典 BFT 能给你什么

Practical Byzantine Fault Tolerance（Castro & Liskov，OSDI 1999）能够容忍 `f < n/3` 的拜占庭节点。协议有三个阶段（pre-prepare、prepare、commit）和两个原语（签名消息、仲裁证书）。在 `n >= 3f + 1` 的诚实或恶意节点中就能就单值达成一致。

这些保证很强，但假设：

1. **故障独立。** 拜占庭节点不会协同。
2. **诚实节点是真正诚实的。** 诚实输出的正确性不是问题；协议只对齐分歧。
3. **问题有客观真值。** 对错误事实的一致仍然是一致。

LLM 智能体违背了上述三点。运行同一基础模型的两个智能体会共享故障。一个“诚实”的 LLM 仍然会产生幻觉。对于模糊的问题，“真理”就是智能体们决定的——不存在外部神谕。

### 三种针对 LLM 的攻击

**Byzantine lie（拜占庭谎言）。** 一个智能体输出故意错误的答案。经典 BFT 在 `f < n/3` 时可以应对。

**Sycophantic conformity（谄媚式顺从）。** 一个智能体在投票前读取其他人的答案，并与最后发言的人保持一致。它不是恶意的，但会与最响亮的声音相关联。经典 BFT 无法阻止这一点，因为该智能体通过了所有签名检查。

**Correlated-error monoculture（相关错误的单一文化）。** 三个智能体共享同一基础模型。它们会产生相同的错误幻觉。多数就错了。经典 BFT 无法帮助，因为这三者“诚实地”达成了一致。

### 2025–2026 年的应对方案

**CP-WBFT**（arXiv:2511.10400）——置信度探针加权 BFT。每个投票者在答案上附带置信度探针（自报概率或独立校准模型的预测）。投票权重按置信度缩放。报告在完全图上提升了 +85.71% 的 BFT 性能。缓解对象：谄媚式顺从（顺从的智能体在其自愿立场上往往置信度较低）。

**DecentLLMs**（arXiv:2507.14928）——无领导式。工作者智能体并行提出，评估者对提案打分，最终答案为被评分位置的几何中位数。当 `f < n/2` 时稳健。缓解对象：拜占庭谎言和相关错误（几何中位数对离群值稳健，会向密集簇靠拢，而不是模型偏差的平均）。

**WBFT**（arXiv:2505.05103）——带层次结构聚类的加权 BFT。投票权重由响应质量加上基于历史学习的信任得分决定。将智能体聚类为 Core 和 Edge；先达成 Core 内的一致，Edge 随后跟随。缓解对象：可扩展性（Core 共识规模小、快速）和部分缓解单一文化（可通过多样性选择 Core）。

### 实证：“Can AI Agents Agree?”（arXiv:2603.01213）

该论文衡量了多种前沿模型之间的标量一致性（LLM 智能体在单一数值问题上的一致性）。结论不太令人安心：

- 即使没有对手，LLM 智能体在许多基准上的标量问题仍有超过 30% 的不同步率。
- 一个采用欺骗性人格的单一智能体可以把 Mixture-of-Agents 的共识从诚实基线拉偏 40+ 个百分点。
- 不同模型之间的分歧率与模型多样性相关——异质集成比同质集成更容易发生不一致（好处：错误不相关），但也更慢趋同（坏处：达成一致的时间更长）。

结论：BFT 给你对齐输出的机制，但不保证对齐的输出是正确的。应当与验证（Phase 16 · 08 角色专职化）、多样性（Phase 16 · 15 辩论变体）和评估者智能体（Phase 16 · 24 基准）结合使用。

### 精简后的核心协议

一个面向 LLM 智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附带置信度探针 c_i，范围在 [0, 1]
3. 聚合器收集来自所有 n 个智能体的 (a_i, c_i)
4. 聚合器按语义聚类分组（等价答案）
5. 聚合器计算每个簇 C 的权重：
     w(C) = sum_{i in C} c_i
6. 若最大权重 max > threshold * sum(c_i)，则 winner = 该最大权重簇
   否则：重试或升级处理
7. 少数簇记录其溯源以便事后审计
```

语义聚类步骤是面向 LLM 的关键。两种答案 “the study reports 4.2%” 和 “4.2% improvement” 属于同一簇。简单的字符串相等检查会漏掉这种情况。在生产环境中，使用廉价的嵌入模型或显式的规范化。

### 阈值调优

`threshold` 参数决定何时接受结果或何时重试。阈值过低：接受了弱多数。阈值过高：几乎永远接受不了。实证范围：对于 `n=5-7`，阈值通常在 0.5–0.67；n 更小时应更高。低于阈值时，升级到人工或者不同的智能体集合。

### 共识无效的场景

- **模糊问题。** 如果问题没有客观真值，共识就是一种观点。就把它称为观点。
- **复合问题。** “写代码并解释它”——其实是两个答案。针对每个子问题分别投票。
- **对抗性多轮。** 如果智能体能观察先前轮次并模仿（见 Du 2023 的辩论），它们会开始相互一致而不顾真相。对轮次数设界（通常 2–3 轮）。

## 构建实现

`code/main.py` 实现了：

- `AgentVoter` — 一个脚本化策略，输出 (answer, confidence)。
- `MajorityVote` — 经典的多数/相对多数投票。
- `CPWBFT` — 带语义聚类的置信度加权投票。
- `DecentLLMs` — 针对有分值提案的几何中位数聚合。
- `Scenario` — 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，并匹配置信度。
3. `monoculture`：三个智能体共享一个错误答案（相关错误），置信度中等。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击，聚合器）-> 最终答案的表格，正确答案高亮。相对多数在 monoculture 场景会失败。CPWBFT 的置信度加权能缓解谄媚攻击。DecentLLMs 的几何中位数在 monoculture 低于一半人口时会向诚实簇拉拢。

## 上线前注意

在部署任何共识机制之前：

- **用至少上面三种模式进行攻击测试。** 你的协议应当可预测地失败，而不是悄无声息地失败。
- **记录每个少数簇及其溯源。** 少数簇是相关错误的早期预警系统。
- **强制限制轮次数。** 不允许“无限争论直到达成一致”——那会奖励谄媚式顺从。
- **将一致性与正确性分离。** 共识输出应交由独立的验证器；验证器应与集合分离。
- **监控一致率。** 一次性飙升意味着从众偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认相对多数在 monoculture 攻击下失败，但当 monoculture 的置信度低于 0.7 时 CPWBFT 部分缓解该攻击。
2. 添加第四种攻击模式：**沉默弃权**——一个智能体拒绝回答（“我不知道”）。每个聚合器应如何处理弃权？实现你的选择。
3. 把语义聚类从字符串规范化替换为嵌入相似度（使用任意开源嵌入模型）。谄媚攻击会发生什么改变？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探针的校准步骤（一个独立的校准模型检查每个智能体自报的置信度）。在 monoculture 场景中测量准确率提升。
5. 阅读 “Can AI Agents Agree?”（arXiv:2603.01213）。复现一个简化的标量一致性实验：三个智能体、一个标量问题、欺骗性人格提示。CPWBFT 或 DecentLLMs 能否检测到该情况？

## 关键术语

| Term | What people say | What it actually means |
|------|----------------|------------------------|
| BFT | "Byzantine fault tolerance" | Castro-Liskov 1999 协议，用于在 `f < n/3` 任意故障下达成共识。 |
| Byzantine | "Any bad behavior" | 一个节点可以撒谎、丢消息、沉默失败——除了安全崩溃以外的任何行为。 |
| Confidence probe | "How sure are you?" | 附加到投票的自报或校准器预测的概率。 |
| Semantic clustering | "Same answer, different words" | 在计票前将等价答案分组。 |
| Geometric median | "Robust center" | 最小化到样本点距离和的点。对离群值稳健，区别于均值。 |
| Monoculture | "Same model, same failures" | 当智能体共享训练数据或基础模型时产生的相关错误。 |
| Sycophantic conformity | "Agreeing with the loud voice" | 智能体的投票会偏向第一个/最响亮发言者。 |
| Core/Edge | "Hierarchical BFT" | WBFT 的划分：先在小规模 Core 内达成一致，然后 Edge 跟随。可界定延迟。 |
