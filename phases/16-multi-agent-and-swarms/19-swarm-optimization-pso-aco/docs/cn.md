# 群体优化用于大型语言模型（PSO，ACO）

> 仿生启发式优化正在在大型语言模型（LLM）领域重获关注。**LMPSO** (arXiv:2504.09247) 使用 PSO，其中每个粒子的速度是一个提示词（prompt），LLM 根据该提示词生成下一个候选；在结构化序列输出（数学表达式、程序）上效果很好。**Model Swarms** (arXiv:2410.11163) 将每个 LLM 专家视为参数流形上的一个 PSO 粒子，在 9 个数据集上对 12 个基线的平均提升达 **13.3%**，仅使用 200 个实例。**SwarmPrompt** (ICAART 2025) 将 PSO 与灰狼算法混合用于提示词优化。**AMRO-S** (arXiv:2603.12933) 借鉴 ACO 的信息素机制用于多智能体路由——**4.7 倍加速**、可解释的路由证据、质量门控的异步更新将推理与学习解耦。本课实现了在提示词参数空间上的 PSO 和在智能体路由上的 ACO，并测量为何这些经典算法适配 LLM 时代，以及何时不适配。

**Type:** 学习 + 构建  
**Languages:** Python（标准库）  
**Prerequisites:** Phase 16 · 09（并行群体网络），Phase 16 · 14（共识与拜占庭容错）  
**Time:** ~75 分钟

## 问题

你的一个提示词在任务评估中得分为 62%。你想改进它。天真的做法是无梯度的人工微调，但这很难扩展。强化学习需要奖励信号和足够的 rollout 来训练。对提示词做反向传播并不现实——提示词是离散字符串，不是可微参数。

经典的仿生启发式优化——PSO 用于连续搜索空间，ACO 用于路径选择——正是为这种场景设计：无梯度、基于群体、每次评估代价低。将它们与 LLM 配对作为无梯度搜索步骤，你会得到一个出人意料的实用优化器。

相同的模式也适用于多智能体系统中的智能体路由。ACO 风格的信息素轨迹记录哪个智能体在何种任务类型上表现更好，路由器可以利用这些轨迹，同时信息素会衰减以便重新发现路线。

## 概念

### PSO 回顾（Kennedy & Eberhart 1995）

Particle Swarm Optimization：在连续搜索空间中的粒子群体。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

其中 `p_best` 是粒子的历史最好，`g_best` 是群体的全局最好，`w, c1, c2` 是惯性 + 认知 + 社会权重，`r1, r2` 是随机因子。

### 将 PSO 应用于 LLM 输出 —— LMPSO

arXiv:2504.09247 将 PSO 改编用于 LLM 生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一个描述如何将当前输出朝向个人/全局最优修改的提示词。LLM 根据速度提示生成新的输出。“速度”的惯性表现为诸如“做小幅增量修改”之类的提示词。

当以下情况成立时效果很好：
- 输出是结构化的（可解析、可评估）。
- 适配度（fitness）是自动的（测试运行、算术评估）。
- 群体规模较小（约 10–30 粒子），因此总的 LLM 调用量可控。

当适配度需要人工审阅时效果不好——每轮的成本会变得不可承受。

### Model Swarms

arXiv:2410.11163 将 PSO 从输出层迁移到模型层。每个“粒子”是一个专家 LLM（参数）。群体通过无梯度更新将参数迁移到集体最好方向。报告显示：在 9 个数据集上对 12 个基线的平均提升为 13.3%，每次迭代仅使用 200 个实例。

关键洞见是 LLM 专家模型在共享参数流形（adapter 权重、LoRA 增量）上已经相互接近。在这个低维子空间上对参数做 PSO 是廉价且有效的。

### ACO 回顾（Dorigo 1992）

Ant Colony Optimization：蚂蚁在图上行进；每条路径有信息素轨迹。蚂蚁的移动概率以信息素强度加权。完成任务的蚂蚁根据解的质量沉积信息素。信息素随时间衰减。

### AMRO-S —— 用于智能体路由的 ACO

arXiv:2603.12933 将 ACO 用于多智能体路由。每个任务类型是一个“目的地”；每个智能体是一条可能的路线。对产生高质量输出的路线增强信息素。关键贡献：

- **可解释的路由证据。** 信息素强度是可人工理解的信号。
- **质量门控的异步更新。** 仅在质量检查通过后更新信息素，从而实现推理与学习解耦。
- **在多智能体路由基准上达成 4.7x 加速。**

质量门控很关键：没有它的话，快速但错误的智能体会积累信息素，系统会陷入糟糕路线。

### 何时使用 PSO / ACO 于 LLM

使用 PSO 的情形：
- 搜索空间是连续的或可映射到连续参数（提示词嵌入、LoRA 权重、数值生成参数）。
- 适配度便宜且自动。
- 群体可以小规模（10–30）。

使用 ACO 的情形：
- 你有路由或路径选择问题。
- 决策随时间能被强化（相同任务类型会重复出现）。
- 你需要可解释的路由证据。

不应使用这两者的情形：
- 适配度需要人工审阅（每次迭代成本太高）。
- 搜索空间是离散且组合性的，PSO 难以覆盖（应使用遗传算法）。
- 实时决策对延迟有严格要求（PSO/ACO 相较于单次启发式收敛慢）。

### 为什么仿生方法仍有优势

基于梯度的方法需要可微信号。LLM 输出和路由决策并不容易被微分。伪梯度方法（强化学习路由器、DPO 风格的提示词调优）能工作，但训练代价高昂。

PSO 和 ACO 只需要一个评估器函数。如果你可以对候选输出或路由决策打分，就能在该空间上优化。这大幅降低了可应用的门槛。

### 实用限制

- 群体预算。N 粒子 × T 迭代 × 每次评估成本。对于每次 LLM 调用约 $0.02 的情景，20 粒子、50 次迭代的 PSO 成本约为 $20。请据此规划。
- 探索与利用的权衡。信息素衰减率与 PSO 惯性权重需要权衡；衰减太快 → 忘记解；太慢 → 陷入早期局部最优。
- 灾难性漂移（catastrophic drift）。如果适配度分布发生变化，这两种算法都可能在收敛后发生偏移并退化（p_best 与信息素过期）。需监控最优适配度的稳定性。

## 实现

`code/main.py` 实现了：

- `LMPSO` —— 在数值提示词参数空间上的 PSO（如 temperature、top_k 权重）。每个粒子的 “LLM 生成” 被脚本化为一个模拟适配度函数。运行 30 次迭代并展示 g_best 的收敛情况。
- `AMRO_S` —— ACO 风格的路由。3 个智能体、4 个任务类型、信息素矩阵、100 次路由任务。打印（task_type → agent choices）随时间的分布来展示轨迹形成。
- 对比：相同任务流上的随机路由与 ACO 路由。测量质量与延迟。

运行：

```
python3 code/main.py
```

预期输出：
- LMPSO：g_best 适配度在 30 次迭代内从随机初始逐步提升至接近最优。
- AMRO-S：信息素表稳定在每个任务类型的正确智能体；ACO 路由在质量上优于随机约 30–40%，并且降低了延迟（更少重试）。

## 使用指南

`outputs/skill-swarm-optimizer.md` 可帮助在 PSO、ACO、遗传算法与基于梯度的优化器之间为 LLM / 智能体优化问题做选择。

## 部署建议

- **从小规模开始。** 10–20 粒子，20–50 次迭代。仅当收敛曲线显示明显收益时再扩大规模。
- **记录信息素或每次迭代的 g_best。** 没有轨迹的群体优化器很难调试。
- **质量门控的更新。** 对于 ACO 路由尤其重要：快速但错误的智能体不应积累信息素。
- **在分布变化时重置衰减。** 当评估分布改变时，老化的信息素会过时；请重置或临时加大衰减率。
- **限制每次迭代的成本。** 输出每次迭代的成本指标。每次迭代花费 $500 且仅提升 0.5% 的 PSO 是不可交付的。

## 关键术语

| Term | What people say | What it actually means |
|------|----------------|------------------------|
| PSO | "Particle Swarm Optimization" | Kennedy-Eberhart 1995。基于群体的无梯度优化器。 |
| ACO | "Ant Colony Optimization" | Dorigo 1992。通过信息素轨迹进行路径/路线优化。 |
| LMPSO | "PSO with LLM generation" | arXiv:2504.09247。速度为提示词；LLM 生成候选。 |
| Model Swarms | "PSO on expert weights" | arXiv:2410.11163。在模型参数子空间上的无梯度更新。 |
| AMRO-S | "ACO for agent routing" | arXiv:2603.12933。在任务类型 × 智能体上的信息素矩阵。 |
| p_best / g_best | "Personal / global best" | 每个粒子的历史最好与群体范围内迄今为止的最好解。 |
| Pheromone | "Routing memory" | 边上的强度；随时间衰减；按质量沉积。 |
| Quality-gated update | "Only learn from good runs" | 信息素沉积以质量检查为条件。 |
| Catastrophic drift | "Distribution shift" | 适配度景观发生变化；旧的 p_best 与信息素变得过时。 |
